## COMMAND SEQUENCE

```powershell

cd path\to\sentinel-prime-network

# Step 2 — Check current status
git status

# Step 3 — Stage files (review before staging)
git add .

# Step 4 — Review what is staged
git diff --cached --name-only

# Step 5 — Commit with SDLC-labeled message
git commit -m "TASK-009: Initial deployment — SPN backend v001 [SDLC:DEPLOY]"

# Step 6 — Push to claude-dev (staging branch)
```

## FILE NAMING STANDARD (from Hub and Spoke workspace)

All new files in the SPN / Hub and Spoke project follow this format:

```
[PROJECT]-[AREA]-[ITEM]_[YYYY-MM-DD]_v001
```

Examples:
- SPN-BACKEND-server_2026-04-26_v001.py
- SPN-DOC-handoff_2026-04-26_v001.md
- HUB-CREATIVE-prompts_2026-04-26_v001.txt

Version increments: v001 → v002 on each meaningful revision.
"v001 = first authentic version" — the baseline for all project artifacts.
